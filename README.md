# Информационная система детского сада

Информационная система для автоматизации деятельности дошкольного образовательного
учреждения. Реализована на основе выпускной квалификационной работы (далее — ВКР) по специальности 09.02.07
«Информационные системы и программирование» (тема: «Разработка информационной
системы для предметной области: детский сад»).

## Возможности

- Учёт воспитанников и их родителей (M:N связь, медицинские сведения в JSON).
- Управление группами с автоматическим контролем предельной наполняемости.
- Кадровый учёт сотрудников с привязкой к группам.
- Ежедневная отметка посещаемости (`/attendance/mark/`).
- Дашборд с ключевыми показателями (`/`).
- Отчёты:
  - сводный по посещаемости группы за период,
  - наполняемость всех групп,
  - кадровый состав (по должностям, средний стаж, доля высшей категории),
  - движение контингента (зачислено / отчислено / выпущено).
- Ролевая модель доступа: **Администратор**, **Заведующий**, **Воспитатель**.
- Готовая Django Admin как основной интерфейс CRUD для всех сущностей.

## Быстрый старт (локально, SQLite)

Без Docker и без PostgreSQL — то, что нужно для разработки и проверки.

```bash
git clone https://github.com/Likuti/Diplom1_1.git
cd Diplom1_1
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env

python manage.py migrate
python manage.py seed_demo            # создаст демонстрационные данные и три учётки
python manage.py runserver
```

Откройте <http://127.0.0.1:8000/>. Демо-учётные записи:

| Логин | Пароль | Роль |
|---|---|---|
| `admin` | `admin12345` | Администратор |
| `head` | `head12345` | Заведующий |
| `teacher` | `teacher12345` | Воспитатель |

## Запуск в Docker (PostgreSQL)

В контейнере дополнительно ставится `psycopg2-binary` через
`requirements-postgres.txt` — локально она не нужна.

```bash
cp .env.example .env                  # отредактируйте при необходимости
docker compose up --build -d
docker compose exec web python manage.py seed_demo
```

Приложение будет доступно на <http://localhost:8000/>.

> Если хотите подключаться к собственному PostgreSQL без Docker,
> установите дополнительный пакет вручную: `pip install -r requirements-postgres.txt`.

## Деплой на Render (бесплатный хостинг)

В корне проекта есть `render.yaml` — Blueprint для Render: один веб-сервис
(Django + gunicorn) и одна managed-PostgreSQL.

1. Зарегистрируйтесь на <https://render.com> через GitHub.
2. <https://dashboard.render.com/blueprints> → **New Blueprint Instance** →
   выберите репозиторий `Diplom1_1`, ветку `main`.
3. Render найдёт `render.yaml`, покажет, что будет создано (web + db) — нажмите **Apply**.
4. Дождитесь деплоя (~5 минут на первый билд). URL будет вида
   `https://kindergarten-is.onrender.com`. Демо-данные (`seed_demo`) добавляются
   автоматически в build-команде — никаких ручных действий после деплоя не нужно.
5. Открывайте URL и логиньтесь демо-учётками.

**Сброс демо-стенда** (очистка БД + повторный посев) — без платного Render Shell:

1. В дашборде → web service `kindergarten-is` → **Environment**.
2. Найдите переменную `DJANGO_RESET_DEMO`, поменяйте значение с `False` на `True`,
   сохраните.
3. **Manual Deploy → Deploy latest commit**. В build-команде выполнится
   `reset_demo` (= `flush` + `seed_demo`).
4. После завершения деплоя верните `DJANGO_RESET_DEMO=False` (иначе при каждом
   следующем деплое БД будет очищаться).

Если у вас платный план с Shell — то же самое одной командой:
`python manage.py reset_demo`.

> Бесплатный план Render «засыпает» после 15 минут без запросов; первый
> заход после простоя занимает ~30 секунд (холодный старт). Перед
> демонстрацией откройте URL за минуту, чтобы прогреть инстанс.
>
> Бесплатная PostgreSQL Render даётся на 90 дней. По истечении нужно
> либо обновить тариф (~$7/мес), либо мигрировать БД, либо переключиться
> на SQLite для демонстрации.

## Соответствие ВКР и сделанные замены

Полный перечень замен и доработок относительно фрагментов кода из ВКР
работы — см. **[docs/replacements.md](docs/replacements.md)**. Кратко:

- `JSONB` → `JSONField` (без потерь функциональности; см. п. 1).
- К `mark_attendance` добавлены валидация даты, `transaction.atomic`, фильтр
  «своя группа» для воспитателя.
- Ролевая модель реализована через `User.role` + декоратор `role_required`,
  а не через жёстко заданные группы Django Auth.
- Добавлены отчёты, которые в ВКР только описаны словами (наполняемость
  групп, кадровый состав, движение контингента).
- Добавлен WhiteNoise для раздачи статики (важно для запуска вне Docker).
